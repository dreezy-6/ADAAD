## Track 1 — GitHub Releases (Immediate, Zero Cost)

### How it works
The `android-free-release.yml` workflow builds a signed APK and attaches it as a
release asset on every `free-v*` tag. Users download the APK directly from
`github.com/InnovativeAI-adaad/ADAAD/releases`.

### Launch steps

```bash
# 1. Tag the release (triggers the workflow automatically)
git tag free-v3.1.0
git push origin free-v3.1.0

# 2. Workflow runs in ~8–12 minutes:
#    ✓ Governance gate (constitutional lint + Android lint)
#    ✓ Signed APK built (community flavor)
#    ✓ APK.sha256 generated
#    ✓ GitHub Release created with installation instructions
#    ✓ F-Droid metadata updated
#    ✓ GitHub Pages PWA deployed
```

### Required GitHub Secrets (Settings → Secrets → Actions)

```
ADAAD_KEYSTORE_BASE64   = base64 -w0 android/keystore/innovativeai-release.jks
ADAAD_STORE_PASSWORD    = <your storepass>
ADAAD_KEY_ALIAS         = adaad-release
ADAAD_KEY_PASSWORD      = <same as storepass for PKCS12>
```

### User installation (sideload)

Users receive these instructions in the release body (auto-generated by CI):

1. Download `adaad-community-3.1.0.apk` from the release assets
2. On Android: **Settings → Apps → Special app access → Install unknown apps**
   → select your browser → enable
3. Open the downloaded APK
4. Android's install dialog appears — tap **Install**
5. Open ADAAD and connect your workspace endpoint

**Minimum Android:** 8.0 (API 26)

## Track 2 — F-Droid (Free, ~1–4 Weeks, Best User Trust)

F-Droid is the canonical open-source Android app repository. No account fee.
No proprietary SDK. Reproducible builds mean users can verify the APK was built
from exactly the source in this repo.

### Sub-track A: Official F-Droid Repository (recommended for reach)

**Estimated time: 1–4 weeks (volunteer review queue)**

```bash
# Step 1: Fork the F-Droid data repository
git clone https://gitlab.com/fdroid/fdroiddata
cd fdroiddata

# Step 2: Copy the ADAAD metadata (already prepared)
cp /path/to/ADAAD/android/fdroid/com.innovativeai.adaad.yml metadata/

# Step 3: Validate locally
fdroid readmeta       # check YAML syntax
fdroid lint com.innovativeai.adaad   # governance-style lint

# Step 4: Open a merge request on GitLab
# Title: "New app: ADAAD — Constitutional Governance"
# Reviewers will:
#   - Verify the source code at the tagged commit
#   - Run a reproducible build and compare APK hash
#   - Check the license (Apache-2.0 ✓)
#   - Check for anti-features (none declared ✓)
```

**F-Droid checklist for this app:**
- [x] License: Apache-2.0 (F-Droid compatible)
- [x] No non-free dependencies in the app module
- [x] No analytics / tracking SDKs
- [x] No non-free network services required (Community tier uses self-hosted endpoint)
- [x] `minSdk 26` — F-Droid supports this
- [x] Metadata file at `android/fdroid/com.innovativeai.adaad.yml`
- [x] Build reproducibility: keystore signing removed in `prebuild` step (F-Droid signs)

### Sub-track B: Self-Hosted F-Droid Repository (instant, full control)

Host your own F-Droid repo on GitHub Pages or any static hosting — zero cost,
available within minutes.

```bash
# Prerequisites
pip install fdroidserver

# 1. Create the repo structure
mkdir adaad-fdroid && cd adaad-fdroid
fdroid init    # generates config.yml and repo/

# 2. Edit config.yml
cat >> config.yml << 'CONFIG'
repo_url = "https://InnovativeAI-adaad.github.io/adaad-fdroid/repo"
repo_name = "ADAAD F-Droid Repository"
repo_description = "Official ADAAD Community APK distribution"
keydname = "CN=InnovativeAI LLC, OU=ADAAD, O=InnovativeAI, C=US"
CONFIG

# 3. Drop the signed APK into repo/
cp adaad-community-3.1.0.apk repo/

# 4. Generate repository index
fdroid update --create-metadata --use-date-from-apk

# 5. Deploy repo/ to GitHub Pages
git init
git add .
git commit -m "Initial ADAAD F-Droid repo"
git remote add origin https://github.com/InnovativeAI-adaad/adaad-fdroid
git push -u origin main
# Enable GitHub Pages for this repo → source: main branch → / (root)

# 6. Users add this repo URL in F-Droid:
#    F-Droid → Settings → Repositories → + Add repository
#    URL: https://InnovativeAI-adaad.github.io/adaad-fdroid/repo
```

**QR code for repository URL** — generate and embed in README:


## Track 3 — GitHub Pages PWA (Instant, Zero Cost, Web-Installable)

The Aponi governance UI is deployed as a Progressive Web App on GitHub Pages.
On Android Chrome, users can tap **"Add to Home Screen"** to install it as a
native-feeling app — no APK download required.

### Enable GitHub Pages

1. GitHub repo → **Settings → Pages**
2. Source: **GitHub Actions** (the workflow handles deployment)
3. The `deploy-pwa` job in `android-free-release.yml` runs automatically on every
   `free-v*` tag

**Public URL:** `https://InnovativeAI-adaad.github.io/ADAAD/`

### PWA install prompt (Android Chrome)
1. User visits the URL in Chrome
2. Chrome shows a banner: **"Add ADAAD to Home Screen"**
3. Tap **Add** → icon appears on home screen
4. Opens in standalone mode (no browser chrome, full-screen governance UI)

### Offline capability
The service worker caches the Aponi shell and constitution rules. Governance
ledger viewing works offline. Mutation proposals require the API endpoint.

## Track 4 — Obtainium Auto-Updates

[Obtainium](https://github.com/ImranR98/Obtainium) lets users track GitHub
Releases and auto-install updates — no app store required.

### For users

```
1. Install Obtainium from F-Droid or GitHub
2. Tap + → Add App
3. Enter: https://github.com/InnovativeAI-adaad/ADAAD
4. Obtainium detects the APK release filter automatically
5. Tap Install — Obtainium checks for updates on your schedule
```

### For one-click import

We ship `android/obtainium.json` in the repo. Users can import it:

```
Obtainium → Import/Export → Import Apps → paste the raw URL of obtainium.json
```

Direct import URL:
```
https://raw.githubusercontent.com/InnovativeAI-adaad/ADAAD/main/android/obtainium.json
```
