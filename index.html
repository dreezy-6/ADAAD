// how it works

Five steps.
One governed loop.

Every epoch runs the same deterministic pipeline. No step is skippable. No gate is optional. No decision goes unrecorded.

01 / PROPOSE 🤖

AI agents compete

Architect, Dream, and Beast independently propose code improvements using Claude. Different philosophies. One shared goal.

02 / EVOLVE 🧬

Genetic selection

Proposals compete in a BLX-alpha genetic algorithm. Best ideas survive. Cross-pollination creates stronger hybrids.

03 / GATE 🛡️

Constitutional gate

16 deterministic rules. One failure = full halt. GovernanceGate is the only surface that can approve a mutation.

04 / RECORD 🧾

Permanent ledger

Every decision is SHA-256 hash-chained, signed, and appended. Tamper the log and the chain breaks immediately.

05 / ADAPT 📈

Weights self-calibrate

Scoring weights shift via momentum gradient descent. What works gains influence. Underperformers decay automatically.

// constitutional rules

The rules aren't
suggestions.

ADAAD operates under a written constitution: 16 rules with binding enforcement. Blocking failures reject mutations outright — no exceptions, no workarounds. The AI agents cannot modify the rules they operate under.


The constitution itself can evolve — but only through a formal process requiring human sign-off and producing its own audit trail.

BLOCKING

ast_validity — Syntax errors are rejected before execution.

BLOCKING

no_banned_tokens — eval, exec, and dangerous primitives are prohibited.

BLOCKING

signature_required — Every mutation must carry cryptographic lineage.

BLOCKING

lineage_continuity — The audit chain cannot have gaps.

BLOCKING

federation_dual_gate — Cross-repo changes need approval in both repos.

WARNING

max_complexity_delta — Large complexity increases are flagged for review.

ADVISORY

reviewer_calibration — Reviewer context captured in every audit record.
