How it works

Five steps. One approval authority.

Every epoch, the system runs a complete mutation cycle. The only thing that can approve a change is the GovernanceGate — architecturally, not by convention.

01
Three AI agents propose independently

Architect, Dream, and Beast — each powered by Claude — propose code improvements without coordinating. They compete, not collaborate.

02
Genetic algorithm tournament

Proposals are scored, crossed-over (BLX-alpha), and ranked. UCB1 bandit allocates exploration budget. Semantic AST diff engine scores risk. Best survive.

03
Constitutional gate — 16 rules

One blocking failure = full halt. AST validity, banned tokens, crypto signature, resource bounds, lineage continuity, entropy budget. No exceptions.

04
Permanent evidence ledger

Every decision is SHA-256 hash-chained, append-only, and deterministically replayable. Alter any past record → chain breaks → pipeline knows immediately.

05
Weights self-calibrate

Scoring weights adapt via momentum gradient descent across epochs. The system learns which improvement types actually work — within constitutional bounds.

Authority invariant GovernanceGate is the only component in the system with mutation approval authority. No agent, config flag, environment variable, or API endpoint bypasses it. This is enforced by architecture, not documentation.

Quick start

Running in 60 seconds

git clone https://github.com/InnovativeAI-adaad/ADAAD.git
cd ADAAD
python onboard.py

Requires Python 3.11+ · pip · git. onboard.py handles everything: environment setup, schema validation, governed dry-run. Every step is idempotent.

from runtime.evolution.evolution_loop import EvolutionLoop
import os

loop = EvolutionLoop(api_key=os.environ["ADAAD_CLAUDE_API_KEY"], generations=3)
result = loop.run_epoch(context)

print(f"Accepted:   {result.accepted_count}")
print(f"Accuracy:   {result.weight_accuracy:.1%}")
print(f"Duration:   {result.duration_seconds:.1f}s")

Honesty

What ADAAD is not

ADAAD does not replace human engineers. It does not guarantee correct code. It does not operate without oversight. It does not learn without constitutional bounds. It does not run silently — every action is logged.

No production deployment happens without explicit human approval. This is not a policy that could be changed by a config flag. It is a structural constraint that the system enforces on itself.
