## Summary

ADAAD v3.1.0 closes Phase 6 ‚Äî Autonomous Roadmap Self-Amendment. The mutation
engine can now propose amendments to this roadmap itself, governed by the same
constitutional gates and human sign-off requirements that govern all mutations.
No roadmap amendment executes without human governor approval recorded in the
evidence ledger.

## Shipped Milestones

### M6-01 ‚Äî RoadmapAmendmentEngine (PR-PHASE6-01)

`runtime/autonomy/roadmap_amendment_engine.py`

Governed propose ‚Üí approve ‚Üí reject ‚Üí verify_replay lifecycle for `ROADMAP.md`
amendments. Authority invariants set `authority_level` as a class constant;
runtime injection raises `GovernanceViolation`. Deterministic replay enforced via
`lineage_chain_hash = SHA-256(prior_roadmap_hash:content_hash)`.

Evidence: 22 replay test scenarios; 100% pass rate.

### M6-03 ‚Äî EvolutionLoop Integration (PR-PHASE6-02)

`runtime/evolution/evolution_loop.py` ‚Äî `_evaluate_m603_amendment_gates()`

Six-gate prerequisite check wired at the post-epoch-N checkpoint in
`EvolutionLoop.run_epoch()`. All gate verdicts are logged to the evidence
ledger regardless of pass/fail. Epoch is never aborted by a gate failure.

| Gate ID | Condition |
|---|---|
| GATE-M603-01 | `epoch_count % trigger_interval == 0` |
| GATE-M603-02 | `health_score(last_10) >= 0.80` |
| GATE-M603-03 | `divergence_count == 0` (if federation enabled) |
| GATE-M603-04 | `prediction_accuracy > 0.60` |
| GATE-M603-05 | `len(pending_amendments) == 0` |
| GATE-M603-06 | `trigger_interval >= 5` |

Environment variable: `ADAAD_ROADMAP_AMENDMENT_TRIGGER_INTERVAL` (default `10`).

Tests: `tests/autonomy/test_evolution_loop_amendment.py` (T6-03-01..13, 13 tests)

### M6-04 ‚Äî Federated Roadmap Propagation (PR-PHASE6-03)

`runtime/governance/federation/mutation_broker.py` ‚Äî `propagate_amendment()`

All-or-nothing propagation with full rollback on any peer failure. Source
approval is recorded as provenance only ‚Äî each destination node runs its own
`GovernanceGate` evaluation and requires its own human governor sign-off.

Ledger events registered: `federated_amendment_propagated`, `federated_amendment_rollback`.

Tests: `tests/governance/federation/test_federated_amendment.py` (‚â•8 tests)

## Constitutional Invariants Added (Phase 6)

| Invariant ID | Rule |
|---|---|
| `PHASE6-AUTH-0` | `authority_level` for amendment proposals is immutable after construction |
| `PHASE6-STORM-0` | At most 1 pending amendment per node at any time |
| `PHASE6-HUMAN-0` | Human governor sign-off non-delegatable for amendment approval |
| `PHASE6-FED-0` | Source-node approval never binds destination nodes |
| `PHASE6-APK-0` | Every APK passes governance gate before signing |
