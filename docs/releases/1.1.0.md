# ADAAD 1.1.0 Release Notes

**Release date:** 2026-03-05
**Phase:** Phase 0 Track A — Security Hardening Complete

---

## Summary

ADAAD 1.1.0 completes Phase 0 Track A hardening. All critical findings (C-01, C-02) from the
deep-dive audit are resolved. The governance environment lifecycle is now fail-closed at every
entry point.

---

## What Changed

### 🔴 Critical Fixes

**C-01 — Boot-Time Environment Validation (PR-HARDEN-01)**
- `env_mode()` in `security/cryovant.py` now raises `RuntimeError("adaad_env_unknown:...")` for any unrecognised `ADAAD_ENV` value instead of silently falling back to `prod`.
- `_validate_boot_environment()` added to `app/main.py`, called as the first line of `main()`.
- Boot fails closed if: env is unset, env is unknown, `CRYOVANT_DEV_MODE` is set in a strict env, or `ADAAD_GOVERNANCE_SESSION_SIGNING_KEY` is missing in a strict env.
- 6 new deterministic test cases in `tests/test_boot_env_validation.py`.
- 3 new test cases in `tests/test_cryovant_strict_env_rejection.py`.

**C-02 — Sandbox Command-Injection Hardening**
- `_DISALLOWED_TOKEN_FRAGMENTS` in `runtime/sandbox/preflight.py` extended with IFS bypass patterns (`$IFS`, `${IFS}`), null-byte variants (`\x00`, `%00`), shell evaluation primitives (`eval `, `exec `, `source `), here-doc operator (`<<`), and newline injection.
- `_DISALLOWED_ENV_KEYS` extended with `LD_LIBRARY_PATH` and `PYTHONSTARTUP`.
- 21 new parametric test cases in `tests/test_sandbox_injection_hardening.py`.

---

### Security Improvements

**Federation Key Pinning (PR-SECURITY-01)**
- `runtime/governance/federation/key_registry.py` implemented and deployed — verifies `key_id` against `governance/federation_trusted_keys.json` before any signature check.
- Caller-supplied public keys are no longer accepted — messages presenting untrusted `key_id` raise `FederationTransportContractError`.
- `docs/governance/FEDERATION_KEY_REGISTRY.md` documents the key lifecycle and rotation runbook.

**SPDX Header Compliance**
- `scripts/check_spdx_headers.py` added to enforce `SPDX-License-Identifier: Apache-2.0` headers on all Python source files.
- 51 previously uncovered files remediated.

## Upgrade Notes

Operators must set `ADAAD_ENV` to one of: `dev`, `test`, `staging`, `production`, `prod`.
Unset or unknown values now cause `SystemExit` at process startup.
In strict environments (`staging`, `production`, `prod`), `ADAAD_GOVERNANCE_SESSION_SIGNING_KEY` must be present.
