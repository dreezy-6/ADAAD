## Maker comment (post immediately after launch):

Hey PH 👋

I built ADAAD because I kept running into the same problem: AI code tools are excellent at generating volume, but terrible at accountability. If your team needs to answer "why did this change happen?" with verifiable proof — not just git blame, but a cryptographic audit trail — most AI dev tools leave you hanging.

**What ADAAD does differently:**

🛡️ **Constitutional governance** — 16 deterministic rules evaluated per mutation. One blocking failure halts everything. The GovernanceGate is the only surface that can approve a change. This is architecturally enforced, not a convention.

🔁 **Deterministic replay** — every decision can be re-run byte-identical. If the replay ever produces a different answer, the pipeline halts. Real auditability, not theater.

🧬 **Genetic algorithm population** — AI proposals compete in a BLX-alpha GA. Best ideas survive. Scoring weights adapt via momentum gradient descent across epochs.

📝 **Roadmap self-amendment** (Phase 6) — the engine can now propose changes to its own development plan. Humans approve. No auto-promote path. It's a constitutional invariant.

📲 **Free Android app** — full governance dashboard, four install tracks, zero cost.

It's completely free and open source. `python onboard.py` gets you running in about 60 seconds.

I'd love to hear from teams dealing with AI governance requirements, compliance engineers, or anyone who's wrestled with "how do we use AI code tools without losing auditability."

Happy to answer any questions about the constitutional model, the determinism architecture, or why three competing AI agents (Architect / Dream / Beast) outperform a single one.
